## Git 工作流

### 分支策略
- main：生产分支
- develop：开发分支
- feature/*：功能分支

### 提交规范
```
feat: 新功能描述
fix: 修复Bug描述
docs: 文档更新
style: 代码风格调整
refactor: 代码重构
test: 测试相关
```

### Pull Request 流程
1. 从 develop 创建功能分支
2. 开发并提交代码
3. 创建 Pull Request 到 main
4. 代码审查
5. 合并到 main
