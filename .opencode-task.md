# 商品管理后台分页 Bug 修复

## 问题描述

商品管理后台无法正确展示商品，接口一次性返回所有数据而不是分页数据。

## 当前状态

- GitHub 代码已更新（包含 range() 分页）
- Vercel 已部署
- 但页面仍然显示所有商品，不是每页 20 条

## 需要检查

1. **Vercel 部署是否正确**
   - 检查部署日志
   - 验证构建产物

2. **代码是否正确**
   - 检查 Admin.tsx 的 loadProducts 函数
   - 确认使用了 range() 方法
   - 确认使用了 count: 'exact'

3. **浏览器缓存**
   - Service Worker 可能缓存了旧代码
   - 需要强制清除

4. **Supabase 查询**
   - 验证 range() 方法是否生效
   - 检查 RLS 策略是否影响

## 文件位置

- src/pages/Admin.tsx
- src/lib/supabaseClient.ts

## 验收标准

1. 点击分类按钮，只加载 20 条商品
2. 控制台显示 `分页范围：0 19`
3. 控制台显示 `加载商品：20 总数：693`
4. 分页导航正常工作

## 调试步骤

1. 在浏览器控制台查看日志
2. 检查 Network 标签的 Supabase 请求
3. 验证请求 URL 是否包含 `&range=0-19`
