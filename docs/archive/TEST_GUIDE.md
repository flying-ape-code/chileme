### 6️⃣ 测试表单验证

**目标**: 验证表单输入验证

#### 6.1 注册表单验证

**测试用例:**

1. **空用户名**
   - 用户名: `(空)`
   - 邮箱: `test2@example.com`
   - 密码: `test123`
   - 确认密码: `test123`
   - **预期**: 显示错误 "请输入用户名"

2. **无效邮箱格式**
   - 用户名: `test2`
   - 邮箱: `invalid-email`
   - 密码: `test123`
   - 确认密码: `test123`
   - **预期**: 显示错误 "请输入有效的邮箱地址"

3. **密码长度不足**
   - 用户名: `test2`
   - 邮箱: `test2@example.com`
   - 密码: `123`
   - 确认密码: `123`
   - **预期**: 显示错误 "密码至少需要 6 个字符"

4. **密码不匹配**
   - 用户名: `test2`
   - 邮箱: `test2@example.com`
   - 密码: `test123`
   - 确认密码: `test456`
   - **预期**: 显示错误 "两次输入的密码不一致"

#### 6.2 登录表单验证

**测试用例:**

1. **空用户名/邮箱**
   - 用户名/邮箱: `(空)`
   - 密码: `test123`
   - **预期**: 显示错误 "请输入用户名/邮箱和密码"

2. **错误的用户名/邮箱**
   - 用户名/邮箱: `wronguser`
   - 密码: `test123`
   - **预期**: 显示错误 "用户名或密码错误"

3. **错误的密码**
   - 用户名/邮箱: `testuser`
   - 密码: `wrongpassword`
   - **预期**: 显示错误 "用户名或密码错误"

**验证点:**
- [ ] 所有表单验证正常工作
- [ ] 错误提示清晰明确
- [ ] 浏览器控制台没有错误

### 7️⃣ 测试数据库验证

**目标**: 验证用户数据正确存储到 Supabase

**步骤:**

1. 注册一个新账号（如 `testuser3@example.com`）
2. 在 Supabase Dashboard 查看：
   - 进入 **Authentication** 页面
   - 查看是否出现新注册的用户
   - 进入 **Table Editor**
   - 查看 `profiles` 表
   - 验证用户信息是否正确（username, email, role）
3. 使用 Supabase SQL Editor 查询：
   ```sql
   SELECT id, username, email, role, created_at
   FROM profiles
   ORDER BY created_at DESC;
   ```

**验证点:**
- [ ] 用户在 auth.users 表中创建成功
- [ ] 用户在 profiles 表中创建成功
- [ ] username 和 email 正确
- [ ] role 默认为 'user'
